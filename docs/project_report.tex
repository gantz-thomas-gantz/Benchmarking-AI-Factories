\begin{abstract}
This project report documents the work carried out by Team~11 as part of the EUMaster4HPC Student Challenge 2025-2026. The objective was to design and implement a modular benchmarking framework for AI workloads on MeluXina; technical design details are presented in the accompanying technical report. Here we describe the results achieved, team organisation, project timeline, and lessons learned.
\end{abstract}

The phased structure of the challenge and the weekly mentoring sessions ensured that all team members remained informed of the project's status and could contribute within their areas of expertise.

\subsection{Collaboration Practices}

The team adopted the following tools and practices for coordination:

\begin{itemize}[noitemsep]
    \item \textbf{GitHub:} Public repository (\texttt{LucaLM02/Benchmarking-AI-Factories}) used for version control, issue tracking, and milestone management.
    \item \textbf{Meetings:} Regular check-ins within the team to discuss progress, blockers, and upcoming tasks, complemented by the weekly mentoring sessions.
    \item \textbf{Issue-driven workflow:} Tasks were tracked as GitHub issues aligned with the challenge phases, allowing transparent progress monitoring.
    \item \textbf{Shared MeluXina workspace:} All team members had access to the project scratch directory on MeluXina, enabling collaborative testing and log sharing.
\end{itemize}
